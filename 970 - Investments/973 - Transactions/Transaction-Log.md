## 使用说明

1. **字段含义**（OpenClaw 需要严格遵守）：
   - **日期**：交易发生日期（YYYY-MM-DD）
   - **操作**：Buy / Sell / Dividend / Deposit / Withdraw / Split / Fee 等
   - **符号**：必须与 Assets 笔记 YAML 中的 `symbol` 完全一致（如 AAPL、BTC、BRK/B）
   - **数量**：买入用正数（如 +50），卖出用负数（如 -30），分红/其他用 0
   - **价格**：单价（分红时可填 0）
   - **货币**：USD / SGD / CNY / USDT 等
   - **总金额**：数量 × 价格（卖出为负数，手续费已单独列出）
   - **平台**：经纪商或交易所名称
   - **手续费**：单独记录
   - **备注**：交易理由、策略或其他说明

2. **推荐用法**
   - **日常**：主要让 OpenClaw 往这个文件追加记录。
   - **归档**：每年年底可以把内容复制到 `2026-Transactions.md`，清空或保留此文件。
   - **查看**：直接打开这个文件就能看到完整交易历史，按日期倒序排列最好。

3. **给 OpenClaw 的追加规则**
   - 在表格最后一行下方新增一行（保持 Markdown 表格格式对齐）。
   - 数量用 + 或 - 符号明确表示。
   - 总金额自动计算（数量 × 价格）。
